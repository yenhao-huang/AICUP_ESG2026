## 兩個主機制（已討論，僅索引）

1. **within_2_years 精度守衛**（L29–37, L41）：定義「主要目標完成年份」、列出非目標日期黑名單、加【長期目標壓制規則】，把「近期日期 → within_2」這個高成本偽陽性壓掉。
2. **already vs between 的「意圖優先」判讀**（L48–61）：先判主訴求是「回報已完成」還是「承諾未來」，把 2024 實績降格為「進度佐證」，破解「看到數字就判 already」。

## 其餘設計技巧（本文件重點）

### A. 任務框定（task framing）

1. **角色 + 封閉前提**（L1–2）：「你是…分類器」＋「**在已知 <data-prompt> 是一個有效 ESG promise 的前提下**」。把上游 gate 的結論寫進前提，模型不必再判斷「這是不是承諾」，任務空間直接縮小到「只判 timeline」，避免越界重判。

2. **封閉輸出詞彙**（L20–24）：「只允許輸出以下四個 label 之一」＋逐一列舉。把開放生成壓成 4 選 1，配合結尾的輸出鎖（見 E）讓下游可直接解析。

### B. 輸入與 context 處理

3. **標籤角色綁定 + 單一主體錨定**（L7–9）：明確規定每個 `<tag>` 區塊的固定角色，並宣告 `<data-prompt>` 是「**唯一主體**」、「timeline 永遠是對這一句下的」。防止模型把 context 當成答案來源、或對頁面上「別句」下判斷。

4. **context 門控（lazy use）+ 觸發清單**（L14–15）：預設只看 `<data-prompt>`；只有「語意不足」才查 context，並**列舉觸發條件**（回指詞「該目標／前述計畫／第二階段／上述承諾」、指涉不清、無年份且模糊）。把「何時該動用 context」變成可檢核的條件，而非模型自由心證。

5. **反洩漏護欄 + 拒絕硬湊**（L16）：context「只用來理解這一句」、「不得把其他承諾/段落的年份當答案」；多年份時「只取與本句直接對應的那一個，**無法對應就回到活動類型判斷，不要硬湊年份**」。後半句給了一條安全退路，避免模型為了用上 context 而強行抓一個年份（這正是放寬 context 後最大的風險）。

6. **時間錨注入**（L18）：「今年是 2024 年，請以 2024 年作為唯一時間錨點」。把推理需要、但 prompt 外不存在的世界狀態常數寫死，讓「兩年內＝2025/2026」這種相對推理變確定。

### C. 決策結構（decision procedure）

7. **優先序決策表 / 先命中先決定**（L27「優先順序由高到低」、L46「由上而下，先命中先決定」）：整個判斷是一條有明確 precedence 的 if-then cascade。把模糊的多類判斷轉成有序規則，**消除平手的不確定性**（同一句符合多條時，順序決定結果）。

8. **樞紐變數的操作型定義**（L29）：不是只給例子，而是先**正面定義**「主要目標完成年份＝明確要『達成/完成/實現某結果』的那個年份」，給模型一個可套用的判準。

9. **具名的衝突裁決規則**（L37【長期目標壓制規則】、L43「取最遠年份」）：兩個訊號衝突時，用一條**有名字**的規則指定誰勝出。具名讓模型更容易「援引並套用」，也方便人事後追蹤是哪條規則在作用。

10. **對高成本類別做不對稱精度守衛**（L41【嚴格條件】）：only within_2 額外加一條「只有 2025/2026 是『完成主要目標』的年份才算」。因為 within_2 偽陽性代價最高（小類、易污染 macro），故對它單獨收嚴，而非全類別一致放寬。

### D. 逐規則技巧

11. **負面清單 / 干擾項黑名單**（L31–35）：明列「**一律不是**主要目標年份」的東西（報告年、基準年、法規生效/起徵年、標準編號年、活動舉辦日、提報日、過渡步驟年）。直接針對模型實測會犯的混淆，先把干擾源排除。

12. **明示 precedence 以壓制貪婪桶**（L58「此規則優先於主題分類」）：模型傾向看到「治理/風險/人才」關鍵字就丟進 between 桶；用一句「此規則優先於主題分類」強制 already 規則蓋過主題關鍵字的貪婪比對。

13. **規則內嵌微範例（mini few-shot）**（L50, L56）：在規則旁直接給具體樣例，如「2024 年受訓時數 7.2 小時」「定期評核比例 100%」「已辦理 N 梯次」。比抽象描述更能對齊模型對該規則的理解。

14. **中性化過載線索詞**（L67）：明講「『持續』『定期』『每年』『已啟動』**本身既不是 already 也不是 between 的充分條件**」。把模型容易過度加權的曖昧詞先拔掉權重，逼它回去看「已完成事實 vs 前瞻承諾」這個真判準。

15. **失敗模式總複習（結尾 anti-pattern recap）**（L66–70 第四步）：把全篇最易錯、最高價值的規則，在結尾以「避免常見誤判」的反面句式再講一次（within_2 守衛、already/between 分界、主題桶不充分、context 不洩漏）。利用 recency + 冗餘，加固最易錯的決策點。

### E. 輸出衛生與排版

16. **輸出格式鎖**（L72）：「僅輸出 label，不要解釋，不要使用 markdown」。保證 `inference/parser.py` 能穩定解析；避免 CoT 文字混入答案。

17. **強視覺分段 + 重點粗體**（全篇 `═══` banner、`**…**`）：用區塊分隔與粗體標出最高槓桿的限制，幫模型在長 prompt 中定位到該套用的規則段落。

## 可遷移檢查清單（寫同類分類 prompt 時）

- [ ] 把上游 gate 結論寫進前提，縮小任務空間（技巧 1）。
- [ ] 綁定每個輸入標籤的角色、錨定唯一判斷主體（技巧 3）。
- [ ] context 採 lazy/門控 + 觸發清單 + 反洩漏護欄 + 拒絕硬湊退路（技巧 4–5）。
- [ ] 注入缺失的世界狀態常數（時間錨）（技巧 6）。
- [ ] 用優先序決策表、先命中先決定（技巧 7）。
- [ ] 對樞紐變數給操作型定義 + 負面黑名單（技巧 8, 11）。
- [ ] 衝突處放具名裁決規則；對高成本類別做不對稱守衛（技巧 9, 10）。
- [ ] 明示 precedence 壓制已知的貪婪 heuristic（技巧 12）。
- [ ] 規則旁放微範例；中性化過載線索詞（技巧 13, 14）。
- [ ] 結尾用反面句式複習最易錯規則（技巧 15）。
- [ ] 鎖死輸出格式以利下游解析（技巧 16）。
